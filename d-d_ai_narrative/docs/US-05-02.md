# US-05-02 — Catalogue campagnes publiques

## Vue d'ensemble

Cette US implémente le catalogue public des campagnes D&D avec filtres par thème et difficulté, recherche par mots-clés et pagination côté serveur. La page `/campaigns` est accessible sans authentification. Le `systemPrompt` est toujours absent des réponses API.

---

## Architecture

```
lib/
├── services/campaign/
│   └── campaignService.ts     ← getPublicCampaigns() avec CampaignFilters + PaginatedCampaigns
│
└── validations/
    └── campaign.ts            ← CampaignFiltersSchema (Zod)

app/
├── api/campaigns/
│   └── route.ts               ← GET /api/campaigns — public, sans auth
│
└── (campaigns)/campaigns/
    └── page.tsx               ← Server Component — SSR initial + Suspense

components/campaigns/
├── CampaignCard.tsx           ← Card cliquable + CampaignCardSkeleton
├── CampaignFiltersBar.tsx     ← Filtres URL-based (thème, difficulté, recherche)
└── CampaignCatalog.tsx        ← Grille + pagination + re-fetch sur searchParams
```

---

## Fonctionnement des filtres

Les filtres sont stockés dans l'URL (searchParams), pas en state local React. Cela permet :
- Le partage de lien avec filtres
- Le bouton retour natif du navigateur
- Le SSR des filtres au premier rendu

**Flux** :
1. Server Component lit `searchParams` → appelle `getPublicCampaigns(filters)` directement
2. `CampaignFiltersBar` met à jour l'URL via `router.push` → déclenche un nouveau rendu serveur
3. `CampaignCatalog` écoute `useSearchParams` → re-fetch `/api/campaigns` côté client pour les mises à jour dynamiques

---

## Route API

```
GET /api/campaigns

Query params (tous optionnels) :
  ?theme=HEROIC|HORROR|MYSTERY|INVESTIGATION
  ?difficulty=EASY|MEDIUM|HARD
  ?search=<string max 100 chars>
  ?isPremium=true|false
  ?page=<int min 1>
  ?limit=<int 1-50, défaut 12>

Réponse :
{
  "success": true,
  "data": {
    "campaigns": [...],   // CampaignPublic[] — sans systemPrompt
    "total": 6,
    "page": 1,
    "totalPages": 1,
    "hasNext": false,
    "hasPrev": false
  }
}
```

---

## Règles métier

- `systemPrompt` absent de `CampaignPublic` — jamais exposé en API publique
- `isPremium=true` → visible dans le catalogue, accès restreint géré en Epic-08
- Pagination côté serveur via `skip/take` Prisma — jamais charger toutes les campagnes en mémoire
- Recherche insensible à la casse sur `title` + `synopsis` via `mode: 'insensitive'`
- L'index `@@index([isPublic, isPremium])` optimise la requête catalogue
