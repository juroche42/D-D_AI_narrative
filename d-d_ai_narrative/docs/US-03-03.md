# US-03-03 — Liste des joueurs en temps réel via SSE

## Objectif

Remplacer la liste statique (host hardcodé) dans le salon d'attente par une liste dynamique mise à jour en temps réel via Server-Sent Events (SSE). Quand un joueur rejoint un salon, tous les clients connectés au même salon voient la liste se mettre à jour instantanément sans rechargement.

Pas de socket.io, pas de Pusher — Web API native uniquement (`EventSource`, `ReadableStream`).

---

## Architecture

```
[Client A] ──EventSource──▶ GET /api/rooms/[code]/events
[Client B] ──EventSource──▶ GET /api/rooms/[code]/events
                                      │
                              sseManager (singleton globalThis)
                              Map<clientId, { roomCode, controller }>
                                      │
           joinRoom() ──broadcastPlayerUpdate()──▶ sseService
                                                        │
                                                   prisma.room.findUnique
                                                   (players avec user)
                                                        │
                                               broadcastToRoom(roomCode, event)
                                                        │
                                         ┌──────────────┴──────────────┐
                                    Client A enqueue             Client B enqueue
                                    data: { players: [...] }     data: { players: [...] }
```

## Flux SSE détaillé

### Connexion initiale
1. Client ouvre `EventSource('/api/rooms/[code]/events')`
2. Serveur vérifie l'auth et que l'user est membre du salon
3. Serveur enregistre le `ReadableStreamDefaultController` dans `globalThis.__sseClients`
4. Serveur envoie immédiatement un snapshot `{ type: 'player_joined', players: [...] }`
5. Serveur programme un keepalive `: keepalive\n\n` toutes les 25s

### Joueur qui rejoint
1. `joinRoomAction()` ou `POST /api/rooms/[code]/join` appelle `joinRoom()`
2. `joinRoom()` crée le `RoomPlayer` en DB
3. `broadcastPlayerUpdate(roomCode, 'player_joined')` est appelé
4. `sseService` recharge la liste complète depuis Prisma
5. `broadcastToRoom()` envoie à tous les clients du salon

### Déconnexion
- `req.signal.abort` → cleanup (clearInterval keepalive + unregisterClient)
- `stream.cancel()` → backup cleanup

### Reconnexion côté client
- Backoff exponentiel : 2s → 4s → 8s → 16s → 30s max
- Abandon après 5 tentatives → message d'erreur + badge "Déconnecté"

## Types SSE

```typescript
type SSEEventType = 'player_joined' | 'player_left' | 'player_updated' | 'room_closed';

interface SSEPlayer {
  userId: string;
  username: string;
  characterId: string | null;
  isReady: boolean;
  isHost: boolean;
  joinedAt: Date;
}

interface SSEEvent {
  type: SSEEventType;
  roomCode: string;
  players: SSEPlayer[];
  timestamp: number;
}
```

## Notes techniques

- **`globalThis.__sseClients`** : le singleton survit aux hot-reloads Next.js dev. En production avec plusieurs workers, chaque worker a son propre state (acceptable pour ce workshop — solution scalable = Redis pub/sub ou Upstash).
- **Keepalive `': keepalive\n\n'`** : c'est un commentaire SSE (ligne commençant par `:`), il ne déclenche **pas** `onmessage` côté client — uniquement pour garder la connexion vivante face aux proxies/load-balancers.
- **`sseManager.ts`** n'a pas `import 'server-only'` car il n'importe rien de Node.js — il est utilisé depuis la route API qui est déjà server-only.
- **`sseService.ts`** a `import 'server-only'` car il importe `@/lib/prisma`.

## Smoke tests

```bash
# 1. Flux SSE brut (avec session cookie valide)
curl -N -H "Accept: text/event-stream" \
  -H "Cookie: authjs.session-token=[TOKEN]" \
  http://localhost:3000/api/rooms/[CODE]/events
# → data: {"type":"player_joined","players":[{"userId":"...","username":"...","isHost":true,...}]}
# → toutes les 25s : ': keepalive'

# 2. Sans session → 401
curl -N http://localhost:3000/api/rooms/[CODE]/events

# 3. Non-membre → 403
# (connecté mais pas dans le salon)

# 4. Test temps réel navigateur
# Onglet 1 : /lobby/[CODE] → badge "En direct" vert
# Onglet 2 : rejoindre via /lobby → entrer le code
# → onglet 1 se met à jour instantanément sans rechargement
```
