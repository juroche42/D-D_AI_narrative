# US-05-04 — Création de campagne avec génération IA du systemPrompt

## Vue d'ensemble

Cette US permet à un utilisateur connecté de créer une campagne D&D personnalisée via un formulaire multi-étapes. Le `systemPrompt` (directives du DM IA) est généré automatiquement par OpenAI GPT-4o-mini à la soumission — stocké en BDD, jamais exposé en réponse API. La campagne est **privée par défaut** (`isPublic = false`).

## Fonctionnement du formulaire

```
/campaigns/new (Server Component)
  └── Redirige vers /login si non connecté

CreateCampaignForm (Client Component)
  ├── Étape 1 — Histoire
  │     title, synopsis, startLocation, mainQuest
  │     Validation react-hook-form + Zod avant de passer à l'étape 2
  │
  ├── Étape 2 — Configuration
  │     theme (grid 2×2), difficulty (grid 3)
  │     minPlayers, maxPlayers, estimatedDuration, isPublic
  │     Au passage étape 2→3 : appel POST /api/campaigns/preview-prompt
  │
  └── Étape 3 — Aperçu IA
        Affiche le systemPrompt généré (lecture seule)
        Récapitulatif des paramètres
        Bouton "Créer la campagne" → POST /api/campaigns
        Redirection vers /campaigns/[id]
```

## Routes API

### `POST /api/campaigns`

Auth requise. Crée la campagne et génère le systemPrompt.

**Body :**
```json
{
  "title": "La Crypte du Roi Oublié",
  "synopsis": "...",
  "startLocation": "Phandalin",
  "mainQuest": "...",
  "theme": "HEROIC",
  "difficulty": "MEDIUM",
  "minPlayers": 2,
  "maxPlayers": 6,
  "estimatedDuration": 120,
  "isPublic": false,
  "isPremium": false
}
```

**Réponse 201 :** `CampaignPublic` — `systemPrompt` absent.

**Erreurs :**
- `401` — non connecté
- `400` — validation Zod échouée
- `400` — `isPremium=true` + `isPublic=false`

### `POST /api/campaigns/preview-prompt`

Auth requise (coût OpenAI). Génère un aperçu du systemPrompt sans persistance.

**Body :** `title`, `synopsis`, `startLocation`, `mainQuest`, `theme`, `difficulty`

**Réponse 200 :** `{ "data": { "systemPrompt": "..." } }`

---

## Génération du systemPrompt

- Modèle : `gpt-4o-mini` — max_tokens: 800, temperature: 0.7
- Coût estimé : ~500 tokens → ~0.001€ par campagne
- Appelé **une seule fois** à la création — résultat stocké en BDD
- `previewSystemPrompt` = alias de `generateSystemPrompt` (même résultat)
- Le prompt méta encode : titre, synopsis, lieu, quête, thème (ambiance), difficulté (style DM)

---

## Règles métier

- `systemPrompt` **jamais exposé** dans les réponses API publiques (`CampaignPublic` l'exclut)
- Campagne **privée par défaut** (`isPublic = false`) — publication future hors scope US-05-04
- `isPremium=true` + `isPublic=false` → rejeté en 400 (cohérence métier)
- Validation par étape dans le formulaire — impossible d'avancer sans champs valides
- `THEME_CONFIG` et `DIFFICULTY_CONFIG` importés depuis `@/lib/constants/campaign`
