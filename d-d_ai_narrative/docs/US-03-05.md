# US-03-05 — État du salon (WAITING → IN_PROGRESS → FINISHED)

## Contexte

Cette US active la gestion de l'état du salon dans le lobby. Le host peut démarrer la partie si au moins 2 joueurs sont présents. Le changement de statut est diffusé en temps réel via SSE à tous les clients connectés.

---

## Ce qui a été livré

### 1. Service `updateRoomStatus`

**Fichier** : `lib/services/room/roomService.ts`

Nouvelle fonction exportée :

```typescript
updateRoomStatus(roomCode: string, userId: string, newStatus: 'IN_PROGRESS' | 'FINISHED'): Promise<RoomPublic>
```

**Transitions autorisées** :
- `WAITING → IN_PROGRESS` (démarrer la partie)
- `IN_PROGRESS → FINISHED` (terminer la partie — US futures)

**Règles métier** :
- 403 si l'appelant n'est pas le host
- 409 si la transition n'est pas dans la liste autorisée
- 422 si < 2 joueurs pour passer en `IN_PROGRESS`
- 404 si le salon n'existe pas

### 2. API endpoint

```
PATCH /api/rooms/:code/status
```

**Corps de la requête** :
```json
{ "status": "IN_PROGRESS" }
```

**Réponse 200** :
```json
{
  "data": { "id": "...", "code": "ABC123", "status": "IN_PROGRESS", ... },
  "message": "Statut du salon mis à jour"
}
```

**Erreurs** :
- `401` — non authentifié
- `400` — body invalide (status absent ou valeur hors enum)
- `403` — non-host
- `404` — salon introuvable
- `409` — transition invalide
- `422` — pas assez de joueurs

### 3. Server Action `startGameAction`

**Fichier** : `app/(lobby)/lobby/actions.ts`

```typescript
startGameAction(roomCode: string): Promise<StartGameResult>
```

Appelle `updateRoomStatus` avec `'IN_PROGRESS'`. Retourne `{ success: true }` ou `{ success: false, error: string }`.

**Fichier** : `hooks/useRoomPlayers.ts`

Nouveau champ dans la valeur retournée :
```typescript
roomStatus: string  // ex. 'WAITING', 'IN_PROGRESS', 'FINISHED'
```

Le hook lit `data.status` de chaque événement SSE et met à jour l'état local.

### 6. Composant `RoomStatusBadge`

**Fichier** : `components/lobby/RoomStatusBadge.tsx`

Props : `status: string`, `size?: 'sm' | 'lg'`

### 7. `RoomLobby` mis à jour

**Fichier** : `components/lobby/RoomLobby.tsx`

- Appelle `useRoomPlayers(room.code)` pour obtenir `players` et `roomStatus`

## Architecture SSE post-US-03-05

```
SSEEvent {
  type: 'player_joined' | 'player_left' | 'player_updated' | 'room_closed' | 'room_status_changed'
  roomCode: string
  players: SSEPlayer[]
  status: string          ← RoomStatus courant du salon
  timestamp: number
}
```

Tous les événements incluent `status`, permettant au client de toujours connaître l'état courant du salon.

## Note sur la redirection `/game/:code`

Lors d'un événement `room_status_changed` avec `status: 'IN_PROGRESS'`, le hook met à jour `roomStatus` (et donc le badge et le message d'attente) mais **ne redirige pas** vers `/game/:code`. Cette page n'existe pas encore (US Epic-05). La redirection sera ajoutée dans une future US.
