# US-06-03 — Sélection de campagne dans le lobby (SSE temps réel)

## Contexte

Le host peut sélectionner ou désélectionner une campagne dans le lobby. La sélection est persistée en BDD (`Room.campaignId`) et diffusée en temps réel à tous les joueurs via SSE (`campaign_selected`). Le bouton "Démarrer" reste désactivé tant qu'aucune campagne n'est sélectionnée.

## Fonctionnement

1. **Host uniquement** : le slot campagne est un bouton cliquable qui ouvre la modal de sélection. Les non-host voient l'état en lecture seule.
2. **Modal** : liste les campagnes publiques + campagnes privées du host (`GET /api/rooms/:code/campaigns?limit=50`). Filtrable par recherche (debounce 300ms, filtre côté backend). Bouton "Retirer" si une campagne est déjà sélectionnée.
3. **Sélection** : appelle `selectCampaignAction(roomCode, campaignId)` → `POST /api/rooms/:code/campaign` → `selectCampaign()` en service.
4. **SSE** : `broadcastToRoom` diffuse `{ type: 'campaign_selected', campaign: { id, title, theme, difficulty } | null }` à tous les clients connectés. Le `systemPrompt` n'est **jamais** inclus dans les events SSE.
5. **Hook** : `useRoomPlayers(roomCode, initialCampaign)` expose `selectedCampaign` mis à jour en temps réel.
6. **`isHost` dynamique** : dérivé de `players` (état SSE live) via `myPlayer?.isHost`, avec fallback sur `room.hostId` pour le rendu initial. Cela permet au nouveau host de voir ses contrôles immédiatement après un transfert de host, sans rechargement de page.
7. **canStart** : `isHost && roomStatus === 'WAITING' && players.length >= 2 && selectedCampaign !== null`.

## API

### `POST /api/rooms/:code/campaign`

**Auth** : requise
**Body** : `{ "campaignId": "string | null" }`
**Réponses** :
- `200` — `RoomPublic` (sans `systemPrompt`)
- `403` — non-host
- `404` — salon ou campagne introuvable
- `409` — salon non en WAITING

### `GET /api/rooms/:code/campaigns`

**Auth** : requise
**Query params** : `?search=&theme=&difficulty=&isPremium=&page=&limit=`
**Réponses** :
- `200` — `PaginatedCampaigns` (campagnes publiques + campagnes privées du host connecté)
- `401` — non authentifié

## Sécurité

- `systemPrompt` n'est jamais exposé dans les réponses API ni dans les events SSE.
- Seul le host peut modifier la campagne sélectionnée.
- La liste des campagnes disponibles inclut uniquement les campagnes publiques et celles créées par le host connecté.
