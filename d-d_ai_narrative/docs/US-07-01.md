# US-07-01 — Synchronisation des votes en temps réel

## Résumé

Implémente la synchronisation temps réel des votes entre joueurs :
- Tous les joueurs d'une même room voient les **mêmes 3 actions** (fix du bug US-06-05)
- Les **compteurs de votes** se mettent à jour en direct pour tous les joueurs
- Chaque joueur peut **changer son vote** (un seul vote par tour)
- Un badge "**Mon vote**" indique l'action pour laquelle le joueur a voté

---

## Architecture

### Flux de données

```
[Client A connecte]
  → GET /api/game/:code/events (SSE persistant)
    → snapshot initial : actions_ready { actions, votes, myVote }

[Client A vote]
  → POST /api/game/:code/vote { actionId }
    → prisma.vote.upsert (1 vote/joueur/tour, changeable)
    → broadcastToGame(roomCode, vote_cast { votes }, perUser → myVote)
      → tous les clients reçoivent les nouveaux compteurs
      → chaque client reçoit son propre myVote
```

### Idempotence des actions (fix bug)

`getOrCreateTurnActions` vérifie la DB avant d'appeler OpenAI :
1. Si des `TurnAction` existent pour ce tour → les retourner directement
2. Sinon → `generateTurnActions` + `saveTurnActions` (une seule fois)

Tous les clients reçoivent donc les mêmes actions, qu'ils arrivent simultanément ou en décalé.

## Contrat API

### `GET /api/game/:code/events`

SSE persistant. Requiert authentification et appartenance à la room.

**Événements reçus :**

| type | contenu | description |
|------|---------|-------------|
| `actions_ready` | `{ actions, votes, myVote, turn }` | Snapshot initial à la connexion |
| `vote_cast` | `{ votes, myVote, turn }` | Après chaque vote (broadcast) |
| `turn_resolving` | `{ turn }` | Quand le DM commence à narrer |

**Keepalive :** commentaire SSE (`: keepalive`) toutes les 25s pour maintenir la connexion.

### `POST /api/game/:code/vote`

Body : `{ actionId: string }`

Codes retour : `200 { ok: true }` / `404` action introuvable / `409` partie non en cours / `422` validation.

## Modèles de données

```
Vote @@unique([gameStateId, userId, turn])
→ 1 vote par joueur par tour, changeable via upsert
→ Cascade delete si GameState supprimé

TurnAction
→ Créées une seule fois par tour (idempotent via getOrCreateTurnActions)
→ Référencées par les votes
```

---

## Hook `useGameEvents`

```typescript
const { actions, votes, myVote, currentTurn, ready, castVote } = useGameEvents(roomCode);
```

- **`ready`** : `true` dès réception de `actions_ready` → phase `voting`
- **`actions`** : `GameActionItem[]` — partagés entre tous les clients
- **`votes`** : `VoteCount[]` — mise à jour en direct
- **`myVote`** : `string | null` — actionId voté par ce joueur
- **`castVote(actionId)`** : POST + attente du broadcast SSE pour màj compteurs

Reconnexion avec backoff exponentiel : 2s → 4s → 8s → 16s → 30s (max 5 retries).
