# US-03-01 — Création et gestion du salon de jeu (Room)

## Objectif

Permettre à un utilisateur authentifié de créer un salon de jeu avec un code d'invitation unique à 6 caractères (`[A-Z0-9]`), d'accéder à son lobby, et d'inviter d'autres joueurs via un lien ou un code.

---

## Architecture

```
lib/services/room/
  roomService.ts        # Logique métier (createRoom, getRoomByCode)
  index.ts              # Barrel export

app/api/rooms/
  route.ts              # POST /api/rooms — endpoint REST

app/(lobby)/lobby/
  page.tsx              # Page liste des salons (Server Component)
  actions.ts            # Server Action createRoomAction()
  [code]/
    page.tsx            # Page lobby d'un salon (Server Component)

  LobbyEntrance.tsx     # UI création/rejoindre (Client Component)
  RoomLobby.tsx         # UI du lobby actif (Client Component)

```

## Modèles Prisma

### Room

| Champ       | Type          | Description                        |
|-------------|---------------|------------------------------------|
| id          | String (cuid) | Identifiant unique                 |
| code        | String unique | Code d'invitation `[A-Z0-9]{6}`    |
| name        | String        | Nom du salon (`Salon de <username>`) |
| status      | RoomStatus    | `WAITING` / `IN_PROGRESS` / `FINISHED` |
| actionMode  | ActionMode    | `VOTE` / `FREE`                    |
| maxPlayers  | Int           | Max 6 par défaut                   |
| hostId      | String (FK)   | Référence vers User (host)         |

### RoomPlayer

| Champ       | Type          | Description                        |
|-------------|---------------|------------------------------------|
| id          | String (cuid) | Identifiant unique                 |
| roomId      | String (FK)   | Référence vers Room (cascade delete) |
| userId      | String (FK)   | Référence vers User                |
| characterId | String? (FK)  | Personnage sélectionné (optionnel) |
| isReady     | Boolean       | Joueur prêt à démarrer            |

## Flux de création d'un salon

```
Client (LobbyEntrance)
  → createRoomAction() [Server Action]
    → auth() — vérif session
    → createRoom(userId, username) [Service]
      1. Vérifie si déjà dans un salon WAITING → 409 CONFLICT
      2. generateUniqueCode() — retry jusqu'à 5 fois
      3. prisma.$transaction → Room + RoomPlayer créés atomiquement
      4. Retourne RoomPublic (DTO sans données Prisma brutes)
    → redirect('/lobby/[code]')
```

### Code d'invitation

- Généré via `customAlphabet('ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789', 6)` (nanoid)
- Alphabet strict `[A-Z0-9]` — pas de caractères ambigus (0/O, 1/I)
- Retry jusqu'à 5 tentatives en cas de collision (quasi-impossible en pratique)

## API REST

### `POST /api/rooms`

Crée un nouveau salon.

**Auth requise** : oui (session JWT)

**Réponse 201** :
```json
{
  "success": true,
  "data": {
    "id": "cm...",
    "code": "ABC123",
    "name": "Salon de ThorinHero",
    "status": "WAITING",
    "maxPlayers": 6,
    "hostId": "cm...",
    "createdAt": "2026-03-03T...",
    "inviteLink": "http://localhost:3000/lobby/ABC123"
  },
  "message": "Salon créé avec succès"
}
```

**Erreurs** :
- `401 UNAUTHORIZED` — utilisateur non connecté
- `409 CONFLICT` — déjà dans un salon WAITING

## Interface `RoomPublic`

```typescript
interface RoomPublic {
  id: string;
  code: string;
  name: string;
  status: string;
  maxPlayers: number;
  hostId: string;
  createdAt: Date;
  inviteLink: string;  // calculé via NEXTAUTH_URL
}
```

## Pages

### `/lobby` — `app/(lobby)/lobby/page.tsx`

Server Component. Vérifie la session → redirect `/login` si absent. Affiche `<LobbyEntrance>`.

### `/lobby/[code]` — `app/(lobby)/lobby/[code]/page.tsx`

Server Component. Récupère le salon via `getRoomByCode(code)`. Affiche `<RoomLobby>` ou `notFound()`.

> **Note Next.js 16** : `params` est une `Promise` → `const { code } = await params`

---

## Composants

### `LobbyEntrance` (Client)

- Bouton **Créer un Salon** → `createRoomAction()` → redirect
- Bouton **Rejoindre** désactivé (US-03-02)
- Gestion d'erreur inline (message sous le bouton)

### `RoomLobby` (Client)

- Code salon affiché en rouge mono + bouton copier (`navigator.clipboard`)
- Lien d'invitation copiable
- Slots joueurs : 1 host (couronne) + N slots vides
- Placeholders : US-06-03 (scénario) + US-04-04 (personnage)
- Bouton **Quitter** → `router.push('/lobby')`
- Bouton **Démarrer** désactivé (en attente des joueurs)
