# US-06-02 — Indexation SRD D&D 5.1 + Service RAG pgvector

## Vue d'ensemble

Cette US implémente le pipeline RAG (Retrieval-Augmented Generation) permettant au Dungeon Master IA de récupérer des règles D&D 5e pertinentes à chaque tour de jeu.

## Architecture

```
Open5e API ──► scripts/indexSrd.ts ──► PostgreSQL vector_documents
                                            │
                                            ▼
              ragService.ts ◄── embed() ◄── OpenAI text-embedding-3-small
                    │
                    ▼
              buildRagContext() ──► System Prompt enrichi
```

## Modèle Prisma

```prisma
model VectorDocument {
  id        String     @id @default(cuid())
  createdAt DateTime   @default(now())
  source    String                          // ex: "open5e:spells:fireball"
  chunk     String                          // texte brut du chunk
  type      MemoryType                      // RULE | LORE | NPC | QUEST | EVENT
  embedding Unsupported("vector(1536)")?   // embedding OpenAI 1536 dims

  @@index([type])
  @@map("vector_documents")
}
```

## Script d'indexation

```bash
pnpm index:srd          # indexe si la table est vide
pnpm index:srd:force    # réindexe (supprime + recrée tous les documents)
```

**Sources Open5e indexées** : spells, monsters, classes, races, sections, magicitems

**Paramètres** :
- Chunk size : ~2000 chars (~500 tokens)
- Overlap : ~200 chars (~50 tokens)
- Batch embeddings : 10 textes par appel, délai 200ms inter-batch
- Modèle d'embedding : `text-embedding-3-small` (1536 dims)

## API RAG

### GET /api/ai/rag
Retourne les statistiques d'indexation.
```json
{
  "success": true,
  "data": {
    "indexed": true,
    "total": 1347,
    "byType": { "RULE": 523, "LORE": 612, "NPC": 212 }
  }
}
```

### POST /api/ai/rag
Recherche par similarité cosinus.
```json
// Body
{ "query": "comment fonctionne l'initiative en combat", "limit": 5 }

// Response
{
  "success": true,
  "data": {
    "count": 3,
    "documents": [
      { "id": "...", "source": "open5e:sections:combat", "chunk": "...", "type": "RULE", "similarity": 0.87 }
    ]
  }
}
```

## Service ragService.ts

| Fonction | Description |
|----------|-------------|
| `searchSimilarDocuments(query, limit?, threshold?)` | Recherche sans filtre de type |
| `searchByType(query, type, limit?, threshold?)` | Recherche filtrée par MemoryType |
| `buildRagContext(situation)` | Recherche parallèle RULE+LORE+NPC, déduplication, formatage |
| `getIndexStats()` | Total + répartition par type |

## Migration

```bash
pnpm db:migrate    # crée la migration add_vector_document
```

Nécessite l'extension pgvector activée : `CREATE EXTENSION IF NOT EXISTS vector;`
