# US-03-02 — Rejoindre un salon via code ou lien direct

## Objectif

Permettre à un utilisateur authentifié de rejoindre un salon existant :
- En saisissant manuellement un code à 6 caractères depuis `/lobby`
- En cliquant sur un lien d'invitation direct `/room/[code]`

## Architecture

### Flux — saisie manuelle (depuis `/lobby`)

```
LobbyEntrance (client)
  → clic "Rejoindre" → affiche l'input code
  → saisie 6 chars → clic "Go" (ou Enter)
  → joinRoomAction(code)          [Server Action]
    → JoinRoomSchema.safeParse()  [validation Zod]
    → joinRoom(code, userId)       [service]
      → findUnique room            [Prisma]
      → checks: 404 / 410 / 422
      → findUnique roomPlayer      [idempotence]
      → findFirst activeElsewhere  [409]
      → create roomPlayer
    → return { success, room }
  → router.push('/lobby/[code]')
```

### Flux — lien direct (`/room/[code]`)

```
/room/[code] (Server Component)
  → auth() → si non connecté → /login?callbackUrl=/room/[code]
  → joinRoom(code, userId)
    → succès → redirect('/lobby/[code]')
    → erreur → redirect('/lobby?error=...')
```

---

## Règles métier

| Cas | HTTP | Comportement |
|---|---|---|
| Salon inconnu | 404 | Erreur affichée inline |
| Salon démarré/terminé | 410 | Erreur "déjà démarré" |
| Salon plein | 422 | Erreur "complet" |
| Déjà dans un autre salon | 409 | Erreur "quittez l'autre salon" |
| Déjà dans **ce** salon | — | Idempotent, retourne le salon sans doublon |
| Non authentifié | — | Redirect `/login` |

## Smoke tests (manuel)

```bash
# 1. Non connecté → 401
curl -X POST http://localhost:3000/api/rooms/ABC123/join | jq

# 2. Preview d'un salon (avec session)
curl http://localhost:3000/api/rooms/[CODE] -b cookies.txt | jq

# 3. Rejoindre (avec session d'un autre user)
curl -X POST http://localhost:3000/api/rooms/[CODE]/join -b cookies2.txt | jq

# 4. Rejoindre à nouveau → idempotent
curl -X POST http://localhost:3000/api/rooms/[CODE]/join -b cookies2.txt | jq

# 5. Code inconnu → 404
curl -X POST http://localhost:3000/api/rooms/XXXXXX/join -b cookies.txt | jq

# 6. Lien direct dans le navigateur
# http://localhost:3000/room/[CODE] → redirect automatique vers /lobby/[CODE]
# sans session → redirect vers /login?callbackUrl=/room/[CODE]

# 7. UI
# http://localhost:3000/lobby → cliquer "Rejoindre" → saisir code → redirect
# code invalide → message d'erreur inline
```

## Notes

- `inviteLink` dans `RoomPublic` pointe désormais vers `/room/[code]` (et non plus `/lobby/[code]`).
  Cela permet aux invités de passer par la route de join automatique avant d'accéder au lobby.
- Les routes API dynamiques (`/api/rooms/[code]`) n'utilisent pas `withErrorHandler` (incompatibilité de signature avec les params Next.js) — elles utilisent try/catch inline avec `AppError` + `ApiResponse.error()`.
