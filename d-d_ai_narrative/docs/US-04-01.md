# US-04-01 — Structure de données : Modèle Character

## Vue d'ensemble

Cette US définit le modèle de données `Character` dans le projet D&D AI Narrative. Un personnage est lié à un `User` et représente l'avatar joué par un utilisateur dans une session de jeu de rôle. Il porte l'ensemble des statistiques D&D 5e nécessaires au moteur narratif IA.

---

## Schéma Prisma

### Modèle `Character`

```prisma
model Character {
  id        String   @id @default(cuid())
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  userId String
  user   User   @relation(fields: [userId], references: [id])

  name  String
  race  Race
  class CharClass

  strength     Int
  dexterity    Int
  constitution Int
  intelligence Int
  wisdom       Int
  charisma     Int

  maxHp      Int
  currentHp  Int
  armorClass Int
  xp         Int @default(0)
  level      Int @default(1)

  @@unique([userId, name])
}
```

### Enums associés

```prisma
enum Race {
```

## Description des champs

### Identité

| Champ       | Type       | Contrainte          | Description                                     |
|-------------|------------|---------------------|-------------------------------------------------|
| `id`        | `String`   | PK, CUID auto       | Identifiant unique généré par Prisma            |
| `createdAt` | `DateTime` | Auto (`now()`)      | Date de création du personnage                  |
| `updatedAt` | `DateTime` | Auto (`@updatedAt`) | Date de dernière modification                   |

### Relation

| Champ    | Type     | Contrainte | Description                                         |
|----------|----------|------------|-----------------------------------------------------|
| `userId` | `String` | FK → User  | Référence vers l'utilisateur propriétaire           |
| `user`   | `User`   | Relation   | Objet User associé (eager/include via Prisma)       |

### Informations générales

| Champ   | Type        | Contrainte | Description                              |
|---------|-------------|------------|------------------------------------------|
| `name`  | `String`    | Unique/User | Nom du personnage (unique par utilisateur) |
| `race`  | `Race`      | Enum       | Race du personnage                       |
| `class` | `CharClass` | Enum       | Classe du personnage                     |

### Statistiques D&D (les 6 caractéristiques de base)

> Valeurs typiques : entre **1** et **20** (max 30 avec magie)

| Champ          | Type  | Description                                                          |
|----------------|-------|----------------------------------------------------------------------|
| `strength`     | `Int` | Force — attaques au corps à corps, port de charges lourdes          |
| `dexterity`    | `Int` | Dextérité — attaques à distance, initiative, discrétion             |
| `constitution` | `Int` | Constitution — points de vie, résistance aux poisons                |
| `intelligence` | `Int` | Intelligence — sorts de magicien, connaissances                     |
| `wisdom`       | `Int` | Sagesse — perception, sorts de clerc/druide, intuition              |
| `charisma`     | `Int` | Charisme — persuasion, sorts de barde/paladin, leadership           |

### Points de vie & combat

| Champ        | Type  | Défaut | Description                                             |
|--------------|-------|--------|---------------------------------------------------------|
| `maxHp`      | `Int` | —      | Points de vie maximum du personnage                     |
| `currentHp`  | `Int` | —      | Points de vie actuels (diminuent lors des combats)      |
| `armorClass` | `Int` | —      | Classe d'armure (difficulté à toucher le personnage)    |

### Progression

| Champ   | Type  | Défaut | Description                                           |
|---------|-------|--------|-------------------------------------------------------|
| `xp`    | `Int` | `0`    | Points d'expérience accumulés                         |
| `level` | `Int` | `1`    | Niveau actuel (1 → 20 dans D&D 5e)                   |

## Contraintes et règles métier

### Unicité
```prisma
@@unique([userId, name])
```
Un utilisateur ne peut pas avoir deux personnages avec le même nom. En revanche, deux utilisateurs différents peuvent avoir un personnage nommé identiquement.

### Relation User → Character
- Un `User` peut posséder **plusieurs** `Character` (relation 1-N)
- Un `Character` appartient à **exactement un** `User`
- La suppression d'un `User` doit être gérée explicitement (pas de cascade configurée)

## Décisions techniques

### Pourquoi `cuid()` et non `uuid()` ?
Prisma recommande `cuid()` pour les identifiants car il est :
- Ordonnable dans le temps (meilleur pour l'indexation BTree PostgreSQL)
- Plus court qu'un UUID v4
- Garanti unique sans coordination entre processus

### Pourquoi un `@@unique([userId, name])` et non un index sur `name` seul ?
Un nom de personnage n'est unique que **dans le contexte d'un utilisateur**. Cette contrainte composite reflète la règle métier exacte et évite de bloquer des noms populaires (ex: "Aragorn") à l'échelle globale.

### Pourquoi stocker `currentHp` séparément de `maxHp` ?
`currentHp` est une valeur **dynamique** qui change au fil des sessions. La stocker en base permet de persister l'état du personnage entre deux sessions de jeu, ce qui est essentiel pour une application de JDR persistant.

### Pourquoi `xp` et `level` sont-ils distincts ?
La montée de niveau en D&D 5e n'est pas automatique : le Dungeon Master peut choisir d'utiliser les "Milestones" (jalons narratifs) plutôt que les XP. Stocker les deux permet de supporter les deux systèmes de progression.
